Figurante Code

Agente de programação com roteamento inteligente entre modelos e destilação progressiva de conhecimento via RAG. Modelos locais ficam mais inteligentes com o tempo, sem pagar por cada resposta.

O problema

Modelos locais de 3B parâmetros são rápidos e gratuitos, mas insuficientes para problemas complexos de código. Modelos de 480B via API são excelentes, mas têm cotas e custo. A pergunta: dá para ter o melhor dos dois mundos?

A ideia central: destilação via RAG

Sessão 1: você faz uma pergunta difícil → roteada para Alibaba 480B → resposta salva no ChromaDB.

Sessão 2: você faz uma pergunta similar → RAG encontra a resposta anterior → modelo local de 3B responde com o conhecimento destilado do 480B. Zero chamadas à nuvem.

Com o tempo, a base de conhecimento cresce e o modelo local resolve cada vez mais perguntas que antes precisariam de API. A cota do 480B é usada apenas para expandir o conhecimento, não para repetir respostas já conhecidas.

Cross-feed bidirecional com o Figurante

Toda resposta do Figurante Code é salva não só na coleção figurante_codigo, mas também na figurante_memoria — o mesmo banco que o assistente principal usa. Isso significa que o Figurante (porta 5000) aprende automaticamente com as interações de código que acontecem no Figurante Code (porta 5001), sem nenhuma ação manual.

O ecossistema completo fica mais inteligente a cada interação.

Por que o qwen3-coder-plus?

480B parâmetros totais com arquitetura MoE — apenas ~30B ativos por token. Eficiência computacional de 30B com capacidade efetiva de 480B. Nos benchmarks de código, supera GPT-4o em várias categorias. A Alibaba oferece cota gratuita de 1-2M tokens nos primeiros 90 dias.
