Case Study: Modelo Preditivo de Risco de Crédito

Um mergulho profundo na construção de uma solução de Machine Learning para reduzir perdas e otimizar a concessão de crédito em uma fintech.

1. Definição do Problema de Negócio

Uma fintech, que chamaremos de "AprovaMais", estava expandindo sua carteira de clientes, mas enfrentava um aumento preocupante na taxa de inadimplência (default). O processo de aprovação de crédito era lento e dependia de regras de negócio que não capturavam a complexidade do perfil dos clientes.

O Desafio Principal:

Como escalar a operação de crédito de forma segura, aprovando clientes com bom potencial de pagamento e recusando aqueles com alto risco, de maneira rápida, consistente e baseada em evidências?

Objetivos Estratégicos:

  • Reduzir a Taxa de Inadimplência: Minimizar as perdas financeiras com maus pagadores.
  • Aumentar a Eficiência Operacional: Automatizar a decisão inicial de crédito para liberar a equipe para casos complexos.
  • Melhorar a Experiência do Cliente: Fornecer respostas de crédito mais rápidas e justas.

2. Análise Exploratória de Dados (EDA)

Com um conjunto de dados históricos de mais de 10 mil clientes, a primeira etapa foi entender as histórias que os dados contavam. A Análise Exploratória foi crucial para descobrir padrões e guiar a construção do modelo.

Principais Descobertas:

Utilizando histogramas, boxplots e um mapa de calor de correlação, identificamos fatores-chave. O mapa de calor, por exemplo, revelou uma forte correlação negativa entre o tempo de emprego e a inadimplência, validando a hipótese de que a estabilidade profissional é um forte indicador de baixo risco.

O mapa de calor ajudou a visualizar as relações entre todas as variáveis simultaneamente.

Gráficos de barras também mostraram que clientes com imóvel próprio tinham uma taxa de inadimplência quase 50% menor em comparação com aqueles que moram de aluguel. Esses insights foram fundamentais para a etapa de engenharia de features.

3. Pré-processamento e Engenharia de Features

Dados brutos raramente estão prontos para um modelo de Machine Learning. Esta etapa consistiu em limpar, transformar e enriquecer o dataset.

Técnicas Aplicadas:

  • Tratamento de Dados Ausentes: Valores faltantes em colunas numéricas (como 'renda') foram preenchidos com a mediana, uma medida robusta a outliers.
  • One-Hot Encoding: Transformei variáveis categóricas (como 'estado_civil' e 'escolaridade') em formato numérico para que o modelo pudesse processá-las.
  • Padronização (StandardScaler): Ajustei a escala de variáveis numéricas como 'idade' e 'renda' para que tivessem a mesma magnitude, evitando que uma variável dominasse as outras.

4. Construção e Avaliação dos Modelos

Com os dados preparados, o próximo passo foi treinar e testar diferentes algoritmos de classificação para encontrar o mais performático.

Modelos Testados:

Dois algoritmos foram selecionados: K-Nearest Neighbors (KNN), como um modelo de baseline simples, e Random Forest, conhecido por sua alta precisão e robustez. Os dados foram divididos em 80% para treino e 20% para teste.

Resultados da Avaliação:

O Random Forest superou o KNN, alcançando uma acurácia geral de 88%. No entanto, em problemas de risco, a acurácia sozinha pode ser enganosa. A Matriz de Confusão foi a ferramenta essencial para uma análise mais profunda.

A matriz mostrou que o modelo foi particularmente bom em identificar corretamente os clientes de 'alto risco' (Verdadeiros Negativos), o que era o objetivo principal para o negócio.

5. Impacto para o Negócio e Próximos Passos

O resultado final não é apenas um código, mas uma solução estratégica com potencial de gerar valor real para a "AprovaMais".

Impacto Potencial:

  • Redução de Perdas: Ao prever com maior precisão os maus pagadores, o modelo pode levar a uma redução estimada de 15-20% nas perdas por inadimplência.
  • Tomada de Decisão Data-Driven: Substitui a subjetividade por um score de risco objetivo, garantindo consistência e justiça nas aprovações.
  • Dashboard de Insights: O projeto culminou na proposta de um dashboard simples onde os resultados podem ser consultados, democratizando o acesso à inteligência gerada.

Próximos Passos:

O trabalho não termina aqui. Os próximos passos incluem o deploy do modelo como uma API para integração com o sistema principal da fintech, monitoramento contínuo de sua performance e o retreinamento periódico com novos dados de clientes.
