Case Study: Análise Preditiva de Risco de Crédito

Utilizando Machine Learning para otimizar a tomada de decisão e minimizar perdas financeiras para a Fintech "AprovaMais".

1. Contexto do Negócio

A "AprovaMais", uma fintech em crescimento, enfrentava um desafio crucial: como aprovar o maior número de empréstimos possível sem aumentar o risco de inadimplência? Decisões de crédito imprecisas estavam resultando em perdas financeiras e na negação de crédito para clientes potencialmente bons. O objetivo era substituir o processo manual e subjetivo por uma solução baseada em dados.

Objetivos do Projeto:

  • Criar um modelo de Machine Learning capaz de classificar novos pedidos de crédito como "Bom", "Ok" ou "Ruim".
  • Identificar as principais variáveis que influenciam o risco de um cliente.
  • Desenvolver um dashboard simples para que a equipe de crédito pudesse consultar os resultados do modelo.

2. Ferramentas e Tecnologias

Para este projeto, utilizei um conjunto de ferramentas padrão da indústria de Ciência de Dados, focando em eficiência e performance.

Python Pandas NumPy Scikit-learn Matplotlib Seaborn Google Colab

3. O Processo: Da Análise à Modelagem

Análise Exploratória dos Dados (EDA)

O primeiro passo foi mergulhar nos dados históricos. Investiguei a distribuição de variáveis como idade, renda, tempo de emprego e histórico de crédito. Um dos primeiros insights foi que a distribuição de renda era assimétrica, o que exigiu uma transformação logarítmica para normalizá-la e melhorar o desempenho do modelo.

Exemplo de gráfico: Distribuição da idade dos clientes, mostrando a concentração de pedidos de crédito.

Pré-processamento e Treinamento do Modelo

Após a análise, os dados foram limpos e preparados: tratei valores ausentes, converti variáveis categóricas (como estado civil) em números e separei os dados em conjuntos de treino e teste. Treinei dois modelos diferentes, Random Forest e KNN, para comparar qual deles conseguia prever o risco com maior precisão.

Avaliação de Performance

O modelo Random Forest apresentou o melhor resultado, com uma acurácia de 88% nos dados de teste. Isso significa que, de cada 100 previsões, o modelo acertou 88. A matriz de confusão abaixo mostra que ele foi especialmente eficaz em identificar os clientes de "Alto Risco", que é o erro mais caro para o negócio.

Matriz de Confusão do modelo, destacando a precisão em cada classe de risco.

4. Resultados e Dashboard Final

O principal resultado foi um modelo funcional que automatiza e melhora a precisão da análise de crédito. Além disso, a análise revelou insights importantes:

  • Tempo de Emprego: É a variável mais forte para prever um bom pagador.
  • Propriedade de Imóvel: Clientes com imóvel próprio apresentaram um risco significativamente menor.

Para tornar esses resultados acessíveis, criei um dashboard interativo (simulado abaixo) onde o analista pode inserir os dados de um novo cliente e receber a classificação de risco instantaneamente, junto com os fatores que mais influenciaram naquela decisão.

Interface do dashboard desenvolvido para consulta rápida da classificação de risco.

5. Conclusão

Este projeto demonstrou com sucesso como a Ciência de Dados pode transformar um processo de negócio crítico. A implementação deste modelo tem o potencial de reduzir a taxa de inadimplência em até 15% e aumentar a eficiência da equipe de análise de crédito. Como próximos passos, sugiro a implementação do modelo em um ambiente de produção real (via API) e o retreinamento periódico com novos dados.
