Figurante

Agente de IA pessoal, offline e com memória semântica persistente. Roda 100% no seu hardware — sem assinatura, sem nuvem, sem nenhum dado saindo da sua máquina.

Ativo / em uso diário Python · Flask Ollama · ChromaDB Dell Latitude 5290 · 8GB RAM
Ver no GitHub Voltar

O problema

Assistentes de IA como ChatGPT e Claude enviam tudo que você digita para servidores externos. Todo documento que você analisa, todo código que você compartilha, toda conversa passa por terceiros. Para uso pessoal e profissional intenso, isso é um problema real de privacidade.

O Figurante foi construído para eliminar esse problema completamente.

A solução

Um assistente pessoal construído sobre dois pilares: Ollama para rodar modelos de linguagem localmente, e ChromaDB para memória vetorial persistente. Todo o tráfego é localhost. Nenhum dado sai da máquina.

Diferencial central: O Figurante tem memória semântica real — não apenas histórico de sessão. Ele lembra de informações de conversas passadas, indexa vetorialmente e as recupera automaticamente quando relevante para a conversa atual.

Como a memória funciona

O sistema de memória é a parte mais interessante tecnicamente. A cada mensagem, o agente faz uma busca semântica no ChromaDB para recuperar contexto de conversas anteriores relevantes:

  • Mensagem do usuário é usada como query vetorial
  • ChromaDB retorna os 5 documentos mais similares (distância coseno < 0.7)
  • Os documentos recuperados são injetados no system prompt como "MEMÓRIA RELEVANTE"
  • O modelo responde com contexto de sessões que podem ter ocorrido dias atrás
  • Após a resposta, o par pergunta/resposta é salvo vetorialmente para uso futuro

Embeddings gerados com all-MiniLM-L6-v2 via ONNX — 100% offline após o primeiro download.

Funcionalidades

  • Dois modos de operação: paz (1.5B, rápido) e ira (3B, preciso) — trocáveis em runtime
  • Memória semântica persistente entre sessões e reinicializações
  • Upload de arquivos: PDF, TXT, CSV, código — conteúdo injetado no contexto
  • Modo Brain para análise profunda de documentos
  • Sistema de regras (regras_figurante.md) — identidade editável pelo próprio chat
  • Streaming com cancelamento real via threading.Event
  • Múltiplas sessões simultâneas com histórico isolado
  • Integração direta com NightStar via painel lateral

Aprendizados

O maior aprendizado foi entender a diferença entre contexto de sessão (efêmero, RAM) e memória semântica (persistente, ChromaDB). O desafio não é salvar — é recuperar as informações certas na hora certa, com relevância suficiente para ser útil sem poluir o contexto do modelo.

Implementar streaming cancelável com SSE e threading.Event também foi mais trabalhoso do que parecia — garantir que threads não ficassem órfãs exigiu atenção ao ciclo de vida completo da requisição.
